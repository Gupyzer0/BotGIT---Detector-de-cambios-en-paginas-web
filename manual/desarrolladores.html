GitHUB

El proyecto posee su propio repositorio en GitHUB donde se podrá obtener la última versión y proponer/pedir cambios.

¿Cómo está estrucuturado el código?

El código está hecho usando la arquitectura MVC (Modelo Vista Controlador) y existe una carpeta para cada elemento. bin es donde se encuentran los elementos del controlador, es también donde se encuentran los widgets de Qt4 usados.
interfaz podemos ubicar las vistas de cada widget definido en el controlador, incluyendo a la interfaz principal.
bdatos se encuentra el archivo qt4BaseDatos.py donde está definida la conexión a la base de datos y las funciones para interactuar con esta, es decir, nuestro modelo. Notese que existe una conexión extra en el archivo main.py que es usada por un hilo de procesamiento diferente al principal ya que EL USO DE UNA CONEXIÓN CREADA DESDE OTRO HILO DE PROCESAMIENTO DIFERENTE AL QUE EJECUTARÁ LAS SENTENCIAS SQL NO ES SEGURO EN QT4
En bdatos también podemos encontrar el archivo palabrasClave.txt donde se encuentran las palabras claves a ser buscadas para detectar desfiguraciones.

Hilos de procesamiento

Se hace uso de hilos de procesamiento a través de un pool de threads para agilizar el procesamiento de las actividades. Notese que cada thread está definido en el archivo "main.py" y que uno de estos, como se mencionó anteriormente, posee una conexión a base de datos propia, en concreto es la clase WorkerAddPortal.

Tiempo de timeout

Puede configurarse en el menú opciones->Tiempo de timeout.
